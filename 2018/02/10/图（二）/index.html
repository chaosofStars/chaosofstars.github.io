Topology Sort（拓扑排序）

概念

拓扑排序常用于解决依赖问题，像软件包之间的依赖。所以有以下两个前提：

  1. 用于有向图
  2. 必须无环

具体思路就是先找到in为0的节点，即没有指向它的路径的节点。然后记下他们，删掉他们通向其他节点的路径。记下删掉后in为0的节点，依次重复，直到遍历过所有节点。

最小生成树算法

概念

最小生成树算法用于找到把所有节点连起来权重最小的路径。具体思路有从点找和从边找两种。

Kruskal算法

k算法用于从边找。把所有边加入一个优先队列，依次从优先队列中取出权重最小的边，判断它两端的点是否属于连通，如果没有就把它记下，然后把两边连通。最后记下的所有边就是结果。

这里连通性的检测需要用到并查集。

Code

Prim算法

prim算法从点开始，把该点的所有边加入到优先队列中。优先队列中弹出权重最小的边，连通两边的点。把连通点的所有边也加入到优先队列中。循环这个过程，直到全部点连通。
